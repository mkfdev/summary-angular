개발 환경 구축

  1. npm / node.js 설치 [설치]
  2. npm으로 Angular CLI 설치
    • [설치법] $ npm install -g @angular/cli@latest
    • Angular CLI 참고사이트
  3. 프로젝트 생성
    • ng new [project-name]
    • 스캐폴딩(파일 구조) 생성
  4. 프로젝트 실행 - [로컬 환경]
    • ng serve 또는 ng serve -o
    • 개발용 서버: http://localhost:4200 접속

CLI로 생성된 폴더와 파일

1. 생성되는 폴더와 파일 리스트
  • 프로젝트를 생성하면 Angular 파일 구조의 스캐폴딩(scaffolding)이 생성
  • node_modules : npm으로 설치되는 패키지가 설치되는 폴더
  • src : 개발자가 실제 소스코드를 추가하는 폴더.
  • angular.json : angular 의 전반적인 설정이 이곳에 들어가며, js, css import, ng 명령어 사용시 생성되는 파일 이름 설정 등을 할 수 있습니다. (angular2에서는 angular-cli.json)
2. src/
  • /app
    • 컴포넌트가 위치하는 폴더
    • 프로젝트 생성시, 루트 컴포넌트(app.component.ts)와 루트 모듈(app.module.ts)가 자동으로 생성됩니다.
    • 필요시 새 컴포넌트를 생성하고, 컴포넌트 안에 여러 개의 컴포넌트를 생성할 수 있습니다.
  • /assets
    • 앱에 사용되는 이미지를 보관하는 폴더
  • /environments
    • 앱의 환경설정을 입력하는 파일이 존재하며, 개발용(~.ts)/배포용(~.prod.ts) 두 가지가 존재합니다.
  • index.html
    • 자바스크립트 파일 로드를 위한 태그나 필요한 매타태그를 작성합니다.

데이터 바인딩

Angular는 단방향 데이터 바인딩과 양방향 데이터 바인딩을 지원합니다.

데이터 바인딩 데이터의 흐름 문법
인터폴레이션 컴포넌트 클래스 ⟹ 템플릿 
{ { expression } }
프로퍼티 바인딩 컴포넌트 클래스 ⟹ 템플릿 [property]=”expression”
어트리뷰트 바인딩 컴포넌트 클래스 ⟹ 템플릿 [attr.attribute-name]=”expression”
클래스 바인딩 컴포넌트 클래스 ⟹ 템플릿 [class.class-name]=”expression”
스타일 바인딩 컴포넌트 클래스 ⟹ 템플릿 [style.style-name]=”expression”
이벤트 바인딩 컴포넌트 클래스 ⟸ 템플릿 (event)=”statement”
양방향 데이터 바인딩 컴포넌트 클래스 ⟺ 템플릿 [(ngModel)]=”property”
1. 인터폴레이션
  • 템플릿 문법으로 표현식의 평과 결과를 문자열로 변환하여 템플릿에 바인딩합니다.

    import { Component } from '@angular/core';
    @Component({
      selector: 'app-root',
      templateUrl:`
        <div class="sm-angular__binding-ex">
            <ul>
              <li>name : {{ name }} </li>
              <li>age : {{ age }} </li>
              <li>admin : {{ admin }} </li>
              <li>address : {{ address.city }} {{ address.contry }} </li>
              <li>gender : {{ gender }} </li>
              <li>say to : {{ sayHi() }} </li>
              <li>Expression : {{ age / 2 }} </li>
              <li>인터폴레이션 : {{ '단방향 데이터 바인딩, 문자열로 변환하여 바인딩' }} </li>
            </ul>
        </div>
        `,
      styleUrls: ['./app.component.css']
    })

    export class AppComponent {
                                
        name = "Angular";
        age = "20";
        admin = true;
        address = {
            city : 'Seoul',
            country: 'Korea'
        }
        sayHi(){
            return 'Hi My name is ${this.name}';
        }

    }
  • name : {{ name }}
  • age : {{ age }}
  • admin : {{ admin }}
  • address : {{ address.city }} {{ address.contry }}
  • gender : {{ gender }}
  • say to : {{ sayHi() }}
  • Expression : {{ age / 2 }}
  • 인터폴레이션 : {{ '단방향 데이터 바인딩, 문자열로 변환하여 바인딩' }}
2. 프로퍼티 바인딩
  • 템플릿 문법으로 표현식의 평과 결과를 HTML 요소의 DOM 프로퍼티에 바인딩합니다.

    import { Component } from '@angular/core';
    @Component({
      selector: 'app-root',
      templateUrl:`
        <div class="sm-angular__binding-ex">
            <input type="text" [value]="inputValue">
            <p [innerHTML]="contents"></p>
            <button type="button" [disabled]="isdisabled">disabled button</button>
        </div>
        `,
      styleUrls: ['./app.component.css']
    })

    export class AppComponent {
                                
        inputValue = 'angular study';
        contents = 'innerHTML property binding text';
        isdisabled = true;

    }

3. 어트리뷰트 바인딩
  • 템플릿 문법으로 표현식의 평과 결과를 HTML 어트리뷰트에 바인딩합니다.

    import { Component } from '@angular/core';
    @Component({
      selector: 'app-root',
      templateUrl:`
        <div class="sm-angular__binding-ex">
            <input type="text" [attr.value]="attrValue">                     
            <table border="1">
                <tr>
                <td [attr.colspan]="number">title</td>
                </tr>
                <tr>
                <td>Angualr</td>
                <td>Javascript</td>
                </tr>
            </table>
        </div>
        `,
      styleUrls: ['./app.component.css']
    })

    export class AppComponent {

        attrValue = 'angular6';
        number = 2;

    }

4. 클래스 바인딩
  • 클래스 attribute에 클래스를 추가/삭제 할 수 있습니다.
  • 단항 클래스 바인딩 / 다항 클래스 바인딩
  • 단항 : 왼쪽에는 [class.반영할 클래스명], 오른쪽에는 참/거짓으로 평가될 수 있는 표현식을 바인딩합니다.
  • 다항 : 좌변에는 [class]를 지정하고, 우변에 클래스의 리스트(문자열)를 바인딩합니다. 기존 class어트리뷰트는 클래스 바인딩에 의해 리셋됩니다.

    import { Component } from '@angular/core';
    @Component({
      selector: 'app-root',
      templateUrl:`
        <div class="sm-angular__binding-ex">

            <div [class.largeText]="isLarge">largeText 스타일 적용된 텍스트</div>
            <div class="smallText" [class]="myClasses">여러 스타일 적용된 텍스트</div>

        </div>

        `,
      styleUrls: ['./app.component.css']
    })

    export class AppComponent {

        attrValue = 'angular6';
        number = 2;

    }
largeText 스타일 적용된 텍스트
여러 스타일 적용된 텍스트
5. 스타일 바인딩
  • 좌변에 반영한 css프로퍼티를 입력하고, 우변에 CSS 프로퍼티의 값으로 평가될 수 있는 표현식을 바인딩한다.

    import { Component } from '@angular/core';
    @Component({
      selector: 'app-root',
      templateUrl:`
        <div class="sm-angular__binding-ex">

            <div [style.font-size.px]="'20'">폰트사이즈 20</div>
            <div [style.background-color]="'#f0f0f0'">배경색</div>
            <div [style.color]="isActive ? '#ff9800':'#1976D2'">폰트컬러 선택</div>

        </div>

        `,
      styleUrls: ['./app.component.css']
    })

    export class AppComponent {

        isActive = false;

    }

6. 이벤트 바인딩
  • 좌변에 반영한 css프로퍼티를 입력하고, 우변에 CSS 프로퍼티의 값으로 평가될 수 있는 표현식을 바인딩한다.

    import { Component } from '@angular/core';
    @Component({
      selector: 'app-root',
      templateUrl:`
        <div class="sm-angular__binding-ex">

            <input type="text" [value]="testName" (input)="setTestName($event)">
            <button (click)="removeTestName()">Remove</button>
            <div>Text : {{ testName }}</div>

        </div>

        `,
      styleUrls: ['./app.component.css']
    })

    export class AppComponent {

        testName = '';

        setTestName(ev){
            this.testName = ev.target.value;
        }

        removeTestName(){
            this.testName = '';
        }

    }
Text : {{ testName }}
7. 양방향 바인딩
  • 뷰와 컴포넌트 클래스의 상태 변화를 상호 반영하는 것.
  • 좌변에는 [(ngModule)](ngModule디렉티브), 우변에는 뷰와 컴포넌트 클래스가 공유할 프로퍼티를 기술합니다.
  • ngModule 디렉티브를 사용하기 위해서는, FormsModule 모듈을 임포트해야 합니다.
  • 양방향 데이터 바인딩은 ngModule 디렉티브만 사용해야 하는 것은 아니며, 커스텀 양방향 데이터 바인딩도 작성할 수 있습니다.

import { BrowserModule } from '@angular/platform-browser';
import { NgModule } from '@angular/core';
import { FormsModule } from '@angular/forms';

import { AppComponent } from './app.component';
import { HomeComponent } from './home/home.component';

@NgModule({
  declarations: [
    AppComponent,
    HomeComponent
  ],
  imports: [
    BrowserModule,
    FormsModule
  ],
  providers: [],
  bootstrap: [AppComponent]
})
export class AppModule { }


    import { Component } from '@angular/core';
    @Component({
      selector: 'app-root',
      templateUrl:`
        <div class="sm-angular__binding-ex">

            <input type="text" [(ngModel)]="longText">
            <div>address: { { longText } } </div>

        </div>

        `,
      styleUrls: ['./app.component.css']
    })

    export class AppComponent {

        longText = '';

    }
